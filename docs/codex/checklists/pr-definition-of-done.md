# PR Definition of Done

Use this checklist before opening or merging a change.

## Checklist

- The change matches the current workspace layout and crate boundaries.
- The code or docs were updated in the smallest reasonable scope.
- Relevant tests were added or updated.
- The narrowest meaningful test command passed.
- Any public behavior change is reflected in the README or docs.
- No secrets, credentials, or private keys were added to the repo.
- Any remaining risk, limitation, or follow-up is called out clearly.

## Documentation Expectations

- Update `README.md` when the project shape or onboarding steps change.
- Update `AGENTS.md` when repository-wide working rules change.
- Update `docs/codex/*` when testing or contribution policy changes.

## Repo-Specific Reminder

- This workspace is still experimental, so it is acceptable to document
  limitations instead of pretending they do not exist.
